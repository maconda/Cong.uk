# Alano Responsive Layout Design

## Goal

在不改变站点既有内容逻辑、页面风格和路由行为的前提下，重构 Alano 个人网站的整体布局与响应式适配，使桌面端、平板端、移动端都具备稳定的导航结构、统一的内容宽度控制和自适应网格。

## Confirmed Constraints

- 保留现有 `index.html` 内容结构和 `src/main.js` 页面切换逻辑。
- 不引入框架，不新增复杂组件。
- 不重做视觉风格，只修布局、比例、断点和网格行为。
- 导航激活态在移动端改为底部指示线。
- 图片比例用 `aspect-ratio`，避免固定宽高。

## Layout Strategy

### Mobile First

以移动端为基础布局：

- `site-shell` 改为单列流式布局。
- `sidebar` 由固定侧栏改为顶部区域。
- `site-nav` 改为横向导航，内容区全宽显示。
- `content` 取消左侧让位，改为自然文档流。

### Tablet

在 `680px` 以上恢复左侧窄栏：

- `sidebar` 重新成为左侧列，但宽度缩小。
- `content` 使用左边距或网格列让出侧栏空间。
- 字号、内边距、块间距整体按比例缩小。

### Desktop

在 `1024px` 以上启用完整左侧固定导航：

- 左侧固定侧边栏。
- 右侧内容区保持稳定最大宽度。
- 网格列数根据可用宽度自动填充，不用固定三列写死。

## Grid Strategy

以下模块统一改为 `repeat(auto-fill, minmax(...))`：

- 图片列表 `mobile-image-list`
- 视频列表 `video-grid`
- 文章列表 `articles-stack`
- 联系方式 `contact-links`

这样不同窗口宽度下自动决定每行数量，减少硬编码断点。

## Sizing Strategy

新增一组响应式变量，用 `clamp()` 管理：

- 页面外边距
- 侧栏宽度
- 标题字号
- 正文字号
- 模块间距

目标是把主要固定 `px` 布局替换为可缩放比例体系，但仅限外层结构和列表，不深入重做每个内容模块的视觉细节。

## File Changes

- `src/styles.css`
  - 追加一段最终布局覆盖层，统一接管外层布局、导航、网格和断点。
- `index.html`
  - 尽量不改；只有在必要时才微调结构。当前方案预计无需结构改动。

## Risks And Mitigation

- 现有样式文件中重复规则很多，容易互相打架。
  - 解决方式：在文件末尾追加高优先级布局覆盖层，不大面积回删旧规则。
- 图片墙、文章详情、视频弹窗存在不同宽度逻辑。
  - 解决方式：只统一外层容器宽度，保留详情页与弹窗内部逻辑。

## Acceptance Criteria

- `>1024px`：左侧固定侧栏，右侧内容区稳定显示。
- `680px–1024px`：侧栏缩窄，内容区不被挤压。
- `<=680px`：顶部横向导航，内容全宽。
- 主要卡片列表使用自适应网格。
- 标题、正文、间距在不同宽度下比例自然。
- 首页、视频、分享、联系页布局表现一致。
