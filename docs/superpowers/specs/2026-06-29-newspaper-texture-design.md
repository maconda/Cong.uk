# Newspaper Texture Design

**Date:** 2026-06-29

**Goal**

在不改变现有网站排版结构、导航逻辑、页面层级和整体气质的前提下，为全站增加统一的轻旧报纸底纹效果，让页面更有纸面感、编辑感和印刷感。

## Scope

- 为全站背景增加统一的轻旧报纸纹理语言。
- 保持现有留白、字体、分栏、模块关系不变。
- 不新增新的页面结构，不重做视觉系统。
- 根据页面类型控制纹理强弱：
  - `分享`、`关于` 页相对更明显
  - `图片`、`视频` 页更弱

## Out Of Scope

- 不把整站改成泛黄复古旧报纸风格。
- 不修改主要内容模块的布局。
- 不改变图片、视频、卡片、导航的核心样式逻辑。
- 不引入新的设计体系或重做主题色。

## Design Direction

采用“统一版，但克制”的方案：

- 整站都有同一类纸面底纹，建立统一视觉语言。
- 纹理不是重做旧，不是大块污渍，不是明显仿古黄纸。
- 纹理主要由以下元素构成：
  - 细微纸张纤维感
  - 非规则轻颗粒
  - 极淡印刷不均匀层
  - 局部轻微明暗起伏

整体观感应接近“高级纸面编辑背景”，而不是“旧报纸收藏页”。

## Page Intensity Rules

### 1. 图片页 / 视频页

- 纹理最弱。
- 目标是让背景不再完全纯净，但绝不影响媒体内容阅读。
- 透明度应明显低于文字页。

### 2. 分享页 / 关于页

- 纹理相对更明显。
- 因为这两页更依赖文字阅读和纸面氛围，适合承接旧报纸气质。
- 但仍不能影响正文清晰度和对比度。

### 3. 联系页

- 介于中低强度之间。
- 保持轻纸感即可，不做强纹理。

## Technical Approach

纹理作为背景层实现，而不是附着在内容层上。

建议通过 CSS 完成：

- 使用 `body::before` 或站点背景容器伪元素承载全局底纹
- 针对不同页面容器叠加局部透明层
- 通过多个渐变和低透明噪点组合模拟纸张感

这样可以保证：

- 易于控制强度
- 不影响现有 DOM 结构
- 回退成本低
- 不破坏组件样式

## Visual Constraints

- 颜色基底仍然保持接近当前的浅色背景
- 不允许明显偏黄褐色整页覆盖
- 不允许大面积脏污斑块
- 不允许纹理压过正文、图片或视频封面
- 不允许让页面从“编辑感”滑向“做旧感”

## Acceptance Criteria

- 全站出现统一但克制的纸面纹理语言
- `分享`、`关于` 页的纸面感明显强于 `图片`、`视频` 页
- 原有页面结构、留白关系和排版逻辑不变
- 文本可读性不下降
- 图片、视频内容不被背景抢视觉

## Files Likely To Change

- `src/styles.css`
- 如有必要：`index.html` 仅用于补充页面标识类，但优先避免修改结构

## Recommended Implementation Order

1. 先做全站极轻统一底纹
2. 再对 `分享`、`关于` 页局部增强
3. 本地预览检查文字和媒体页面是否被干扰
4. 再微调透明度和纹理密度
